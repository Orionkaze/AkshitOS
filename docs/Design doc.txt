📐 AKSHITOS DESIGN DOCUMENT
Inspired by BizNest – Workflow Management System

Prepared for UI & Frontend Implementation

1. DESIGN PHILOSOPHY

AkshitOS is a developer productivity dashboard, not a gaming UI or pastel to-do list. It should:

✅ Feel structured and modular
✅ Be minimal yet expressive
✅ Prioritize clarity over ornamentation
✅ Use data visualization where applicable
✅ Be consistent across modules

Aesthetically, we borrow from BizNest:

✔ Sidebar + top bar
✔ Card panels with crisp hierarchy
✔ Clear:charts, clean grids
✔ Breathable spacing
✔ Intuitive input + feedback

2. COLOR SYSTEM

Dark theme primary | subtle bright accents | readable neutrals

🎨 Core Palette
Token	Purpose	Example

accent-secondary	Graph highlights	#3FA7FF
alert-error	Errors	#E04848
alert-warning	Warnings	#FFAA00

Inspired by BizNest’s contrast and hierarchy — deep surfaces with distinct accent colors for focus.

3. TYPOGRAPHY

Use Inter (or Poppins) for professional clarity.

Style	Size	Weight	Usage
H1	28px	700	Main Page Titles
H2	22px	600	Section Titles
H3	18px	600	Card Titles

Spacing:

Letter spacing normal

Clear hierarchy — no confusion.

8. INTERACTION & MOTION

Motion must be reserved, not distracting.

Rules:

Buttons: scale 1.03 on hover

Cards: fade in on mount

Page transitions: cross-fade

Chart animations: semi-slow ease

Toasts: slide from top

No blank screens.

13. ERROR MESSAGES & LABEL GUIDE

Standard messaging patterns:

Required Field: “This field is required.”

Invalid Entry: “Please enter a valid …”

Save Success: “Saved!”

Delete Confirm: “Are you sure? This action cannot be undone.”

Unauthorized: “Please login to continue.”

Consistency matters.
